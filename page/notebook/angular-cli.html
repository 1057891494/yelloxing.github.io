Angular 快速开始
Angular CLI是一个命令行界面工具，它可以创建项目、添加文件以及执行一大堆开发任务，比如测试、打包和发布。
开始项目前，你需要先安装node和npm，然后执行npm install -g @angular/cli安装Angular CLI。
感性认识
新建一个angular项目，其中--skip-install表示node包先不安装，我们接着使用cnpm install安装会快多了
ng new newApp --skip-install
此时项目已经建立好了，可以cd newApp进入项目，然后使用下面命令启动：
ng serve --open
现在来看看ng命令帮助我们生成了什么，你可以不知道全部，不过先记住下面几个个人感觉重要的：
  • src：应用代码存放的地方
  • src/app：你的代码主要存放的地方，这样说也许不合适，不过你开发的时候，大部分时间都是在修改这里的代码
  • src/assets：图片等存放的地方，构建时会复制到发布包里
  • src/main.js：你基本不会修改它，它是程序的主入口
  • src/styles.css：特别用的样式写在对应的地方，后面会说，对于公共的样式就会写在这里
  • karma.conf.js：给Karma的单元测试配置，当运行ng test时会用到它
刚刚说了，src/app是你经常修改的地方，里面的文件没有介绍，后面会依次涉及。
组件
自定义组件分为三部分：组件主体(这里是test.component.ts)+组件模板(这里是test.component.html)+组件样式(这里是test.component.css)
当前你也可以全部写在一个文件里面，写在组件主体里面
组件主体大致包括三部分：
  • 引入必要的注释对象（这里只是举例，就引入了这一个）：import { Component } from '@angular/core';
  • 修饰组件：@Component({ selector: 'test-component', templateUrl: './test.component.html', styleUrls: ['./test.component.css'] })
  • 定义组件类:export class TestComponent {data = '自定义组件测试数据';}
组件模板和组件样式就是大家写过angularjs的人很熟悉的了，最后使用的时候会看见一行红色的文字
注意：还需要在根模块app.module.ts里面声明一下，具体查看别的组件即可。
